在 GoLand 使用 go-lint 进行代码规范检查

Golint 是一个源码检测工具用于检测代码规范，不同于 gofmt 的格式化方式还增加了一些推荐的代码语法规范，然而每次命令行使用其处理文件或是目录还是很麻烦的，但好在 GoLand 提供了比较人性化的使用方式，下面记录一下配置过程

  1. 安装 go-lint 这时 go-lint.exe 应该会出现在 $GOPATH/bin 下，没有也不要慌，后边还有其他方式
  2. 去插件市场安装 Go Linter
  3. 然后到 Tool -> Go Linter 下处理具体的配置，如果第一步安装没有的后可以使用 GoLand 提供的下载路径也就是点击 Get Latest
  4. 然后到 Tool -> External Tools 下添加，如过文件路径中有空格记得在 $FilePath$ 两边加引号
  5. 这样就可以了，可以在 KeyMap 中为其配置快捷键，方便使用，我是用的 Ctrl + Shift + Alt + 空格
