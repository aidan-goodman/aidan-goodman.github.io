在阿里云薅羊毛买了台 60 块的服务器，2 核 2G 用了两个周了，感觉没有 GitHub 学生包里的微软服务器香，不过远程服务器的使用还是都差不多的，简单记录一下我的使用心得

买完之后进行简单的设置，主要就是域名绑定和防火墙，域名申请一个然后备案就可以了，防火墙可以打开所有端口（0~65535），也可以用一个开一个

Terminal 远程连接

然后就是远程连接，浏览器可以直接进行远程连接的操作，但用起来也不方便，如果使用客户端需要先设置 Root 密码，我这边没有使用 XShell 之类的进行连接，直接用 Windows terminal 感觉就足够方便了具体连接步骤：

  1. 首先打开电脑的 OpenSSH 功能（在应用的可选功能里面）
  2. 然后直接在命令行（cmd, powershell）执行 ssh -p 22 user@IP 就可以使用了
    -p 是指定 SSH 服务的端口号，默认为 22，然后指定远程的用户名和公网 IP 地址就可以
  3. 修改配置文件进行添加服务，这样每次打开就可直接使用
  1. 可选——免密登录，可以直接使用 Git-Bash 执行 ssh-keygen 在用户目录下生成公私匙，然后将 id_rsa.pub 中的内容复制到服务器的 ~/.ssh/authorized_keys 下，没有就创建文件，权限设置为 600 就可以

文件传输

如果是客户端可以简单使用拖拽的方式进行本地和服务器之间的文件传输，如果不想这么麻烦可以直接使用 scp 命令是实现

VScode 远程开发

  1. 安装 Remote Development 扩展包
  2. 在右侧工具栏的远程资源管理器中添加 SSH Targets 然后连接就 OK
  3. 可以在使用 Docker 时直接修改挂载文件
