SpringBoot 整合 Redis

在 Java 程序中操作 Redis 主要有两种方法：Redis 官方提供的 Jedis；SpringBoot 提供的 starter-redis

虽然在 SpringBoot 2. X 之后内置使用的是 Lettuce，但这里还是全部总结一下，方便后续开发直接从这 CTRL+C

不管 Jedis 还是 Lettuce 本质上都是一个 redis-cli，使用这两个工具可以直接连接 redis-server 进行操作，至于二者的区别如下：

  • Jedis 对 Redis 是进行直接连接的，在多线程环境下是非线程安全的，这个时候只有使用连接池，为每个 Jedis 实例增加物理连接
  • Lettuce 的连接是基于 Netty 的，连接实例（StatefulRedisConnection）可以在多个线程间并发访问，因为 StatefulRedisConnection 是线程安全的，所以一个连接实例（StatefulRedisConnection）就可以满足多线程环境下的并发访问，如果一个连接实例不够也可以按需增加连接实例

Jedis

在 Java 中，Redis 对应于 Jedis 就相当于关系数据库对应于 JDBC，只需要导入 jar 包就可以直接进行使用，为了方便对象转字符串，一般搭配 fastjson 进行使用

然后就可以直接执行操作，jedis 对象的方法是和 Redis API 相对应的，使用起来也没有什么成本

因为 Jedis 对象并不是线程安全的，在多线程下使用同一个 Jedis 对象会出现并发问题。为了避免每次使用 Jedis 对象时都需要重新构建，Jedis 提供了 JedisPool

redis-starter

对于 RedisTemplate 来说，有 spring-boot-starter-data-redis 隔离了 Jedis 和 Lettuce 的差异性，具体使用时没有任何感觉，但和原生的 Jedis 操作还是有一定区别，可定制化也更高了，和其他的 SpringBoot 整合组件类似，都是导入依赖 ==> 文件配置 ==> 开箱使用
