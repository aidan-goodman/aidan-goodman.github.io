WSL2 中进行 Nacos 的安装和使用

之前写过 WSL2+Ubuntu 能够很好的搞定一些这样那样的开发问题，之前学习 Redis 时候也是一直在 WSL 上进行练习，结果这次安装 Nacos 时候却出了一些问题其实主要就是 Ubuntu 和其他版本的 Linux 确实不太一样，这里大致梳理一下正确的安装路线和使用

首先看官网的第一句话

Nacos 依赖 Java 环境来运行。如果您是从代码开始构建并运行 Nacos，还需要为此配置 Maven 环境

WSL 上并没有安装 JDK，所以这里先安装一下 JDK，至于 Maven 我们在 Windows 上写代码的时候只需要调用 Nacos 的服务即可，所以没必要重复安装了（反正不在 WSL 里写代码）

将下载的 Nacos 压缩包放到 /opt 目录下解压，可以先进入 /nacos/bin 执行 sudo bash startup.sh -m standalone 进行单机模式的运行，然后在浏览器进行查看， url = http://localhost:8848/nacos/index.html

网上大部分的教程都是用 sh startup.sh -m standalone 的方式执行，但如果我们在 Ubuntu 中也这么干就会出现脚本错误，这一点官方文档上也有指出
